Song Table
=======================================
To find the pointer to the song table, the following instructions are used:
	add a
	ld e, a
	ld d, 0
	ld hl, SongTab
	add hl, de
	ld a, [hl+]
	ld h, [hl]
	ld l, a


Which equals the following in hex:
87 5F 16 00 21 ?? ?? 19 2A 66 6F AF E0

Alternate for Ayakashi no Shiro:
21 ?? ?? 87 CF 2A 66 6F AF E0

Alternate for Rentaiou/The Shinri Game 1/2:
17 5F 16 00 21 ?? ?? 19 D1 2A 47 2A E5 6F

At least usually, the song table seems to be immediately followed by the sound effects tables (see below).

Song Format:
=======================================
Each song starts with four 16-bit pointers to the sequence for each channel. Following next is the sequence data consisting of the following commands:

00-BF = Play note with length
	Format: xxxxyyyy (X = note (C-B), Y = length (decoded with function, see below))
C0 = Set tempo
	Parameters: xxxxxxxx (X = tempo - raw TMA value)
C1 = Set duty and length (NRx1)
	Parameters: xxxxxxxx (X = raw NRx1 value)
C2 = Set volume/envelope (NRx2)
	Parameters: xxxxxxxx (X = raw NRx2 value)
C3 = Jump to position
	Parameters: xxxxxxxx xxxxxxxx (X = position)
C4 = Set panning (NR51)
	Parameters: xxxxxxxx (X = raw NR51 value)
	(NOTE: This command is only available on channel 1.)
C9 = Return from macro (subroutine)
CD = Go to a macro (subroutine)
	Parameters: xxxxxxxx xxxxxxxx (X = position)
CE = Stop the current channel
Dx = Delay the current note for set length
	Parameters: CCCCxxxx (C = command, x = length (decoded with function, see below))
Ex = Key off the channel for set length
	Parameters: CCCCxxxx (C = command, x = length (decoded with function, see below))
Fx = Set the current octave
	Parameters: CCCCxyyy (C = command, y = "relative mode" flag, x = octave number))
	(If X (relative mode) is set, then the octave will be controlled relative to the current octave.
	0-3 is positive, while 4-7 is negative.
	For example, an octave value of 1 means +1 octave.)
	(NOTE: This function is not available on channel 4.)

To decode note lengths, the following system is used:
	----xyyy (X = "half flag" - multiply value by 1.5, Y = shift count (right))
Using this system, the values of lengths are equal to as follows:
	$x0 -> 128 ticks
	$x1 ->  64 ticks
	$x2 ->  32 ticks
	$x3 ->  16 ticks
	$x4 ->   8 ticks
	$x5 ->   4 ticks
	$x6 ->   2 ticks
	$x7 ->   1 ticks
	$x8 -> 192 ticks
	$x9 ->  96 ticks
	$xA ->  48 ticks
	$xB ->  24 ticks
	$xC ->  12 ticks
	$xD ->   6 ticks
	$xE ->   3 ticks
	$xF ->   1 ticks

Sound Effects
=======================================
In many (most?) games including Mega Man 2, sound effects can use either channel 2 or 4, or both. However, Blaster Master additionally supports channel 3. Sound effects are loaded from separate tables for each channel of a sound effect. In at least Mega Man 2, these use a table of "priorities" that are used to prioritize one sound effect playing over another, similar to the priority flags in the first Mega Man game's sound engine. This appears to be absent from Blaster Master.

Sound effects are an array of 4-byte sequences as follows:
[Repeat]
1. NRx1 value (Channel length/duty)
2. NRx2 value (Volume/envelope)
3. NRx3 value (Period high/control)
4. NRx4 value (Period low)

In Mega Man 2, these sequences end with value 00. However, in Blaster Master, these sequences instead end with value FF.
